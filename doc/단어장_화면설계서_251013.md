1. 화면 설계 개요
1.1 설계 목적
사용자가 직관적으로 사용할 수 있는 UI/UX 설계 및 화면별 기능 정의
1.2 화면 구성 원칙

직관성: 3클릭 이내 모든 기능 접근
일관성: 동일한 UI 패턴 반복 사용
피드백: 모든 액션에 즉시 반응
최소화: 학습에 집중할 수 있도록 불필요한 요소 제거

1.3 주요 변경 사항 (v1.1)

✅ 메인 윈도우: 탭 기반 네비게이션으로 변경
✅ 플래시카드: 플립 카드 애니메이션 적용
✅ 단어 목록: 메모 + 최종학습일 + 오답률 컬럼 추가
✅ 시험: 설정 → 진행 → 결과의 3단계 화면 전환

1.4 전체 화면 구조
메인 윈도우 (QTabWidget)
├── [대시보드] 탭
├── [단어관리] 탭
│   ├── 단어 목록 (메인)
│   ├── 단어 추가 다이얼로그
│   └── 단어 수정 다이얼로그
├── [학습] 탭
│   ├── 학습 설정 화면
│   └── 플래시카드 화면 (플립 방식)
├── [시험] 탭
│   ├── 시험 설정 화면
│   ├── 시험 진행 화면
│   └── 시험 결과 화면
├── [오답노트] 탭
└── [통계] 탭

11. 화면별 상세 인터랙션 정의
11.1 단어 목록 화면 인터랙션
마우스 인터랙션
단일 클릭 (행)
  → 해당 행 선택 (배경색 변경)

더블 클릭 (행)
  → 단어 수정 다이얼로그 열기

우클릭 (행)
  → 컨텍스트 메뉴 표시
    ├─ 수정
    ├─ 삭제
    ├─ 즐겨찾기 토글
    ├─ 이 단어로 복습하기
    └─ 상세 정보 보기

클릭 (즐겨찾기 아이콘)
  → 즐겨찾기 토글 (☆ ↔ ★)
  → 즉시 DB 저장

클릭 (컬럼 헤더)
  → 해당 컬럼 기준 정렬
  → 재클릭 시 오름차순/내림차순 토글
  → 정렬 방향 화살표 표시 (↑/↓)

드래그 (체크박스 선택 후)
  → 다중 선택 가능
키보드 인터랙션
↑/↓ (방향키)
  → 이전/다음 행 선택

Enter
  → 선택된 행 수정 다이얼로그 열기

Delete
  → 선택된 행 삭제 (확인 다이얼로그)

Ctrl+A
  → 전체 선택

Ctrl+C
  → 선택된 단어 클립보드 복사

Ctrl+F
  → 검색 입력창에 포커스

F2
  → 빠른 편집 모드 (인라인 편집)

Space
  → 체크박스 토글
필터/정렬 인터랙션
필터 드롭다운 변경
  → 즉시 목록 재로드
  → 하단 상태바에 "필터링됨: X개" 표시

정렬 드롭다운 변경
  → 즉시 목록 재정렬
  → 애니메이션: 0.2초 페이드

검색어 입력 (실시간)
  → 300ms 디바운스 후 필터링
  → 일치하는 단어 하이라이트

11.2 플래시카드 화면 인터랙션
카드 플립 애니메이션 상세
python# 플립 애니메이션 파라미터
duration = 300  # ms
easing = QEasingCurve.InOutQuad

# 앞면 → 뒷면
rotation: 0° → 90° (전반부 150ms)
  → 내용 교체
  → 90° → 180° (후반부 150ms)

# 시각 효과
- 회전 중 약간 확대 (scale: 1.0 → 1.05 → 1.0)
- 그림자 효과 증가
자동 진행 로직
정답/오답 버튼 클릭
  ↓
피드백 표시 (2초)
  ├─ 정답: 초록색 배경, ✓ 아이콘
  └─ 오답: 빨간색 배경, ✗ 아이콘
  ↓
응답 시간 기록 (DB)
  ↓
2초 후 자동으로 다음 카드
  ├─ 페이드 아웃 (0.2초)
  ├─ 다음 카드 로드
  └─ 페이드 인 (0.2초)
타이머 동작
제한 시간 설정 시:
  - 매초 카운트다운
  - 남은 시간 3초 이하: 빨간색 표시
  - 시간 종료: 자동 오답 처리 + 다음 카드

무제한 시간 설정 시:
  - 경과 시간만 표시
  - 시간 제한 없음

11.3 시험 진행 화면 인터랙션
문제 간 이동
[다음] 버튼 클릭
  → 현재 답안 임시 저장
  → 다음 문제로 이동
  → 이전 답안 있으면 표시

[이전] 버튼 클릭
  → 현재 답안 임시 저장
  → 이전 문제로 이동
  → 답안 복원

[건너뛰기] 버튼 클릭
  → 답안 저장 안 함
  → 다음 문제로 이동
  → 하단 진행 표시에 ○ 유지
하단 문제 번호 네비게이션
문제 번호 클릭
  → 해당 문제로 즉시 이동
  → 현재 답안 임시 저장

색상 표시:
  ✓ (초록): 답변 완료
  ○ (회색): 미답변
  ● (파랑): 현재 문제
자동 저장
매 30초마다:
  - 현재까지 답안 자동 저장
  - 상태바에 "자동 저장됨" 표시 (3초간)

프로그램 비정상 종료 시:
  - 재시작 후 이어하기 옵션 제공

  - 컬럼 부족 → "영어, 한국어 컬럼이 필요합니다"

14. 접근성 고려사항
14.1 키보드 네비게이션
Tab 키
  → 다음 컨트롤로 포커스 이동
  → 포커스 표시: 파란 테두리

Shift+Tab
  → 이전 컨트롤로 포커스 이동

Enter
  → 버튼 활성화
  → 입력 완료

Esc
  → 다이얼로그 닫기
  → 작업 취소
14.2 시각적 피드백
모든 클릭 가능 요소:
  - 호버 시 커서 변경 (pointer)
  - 호버 시 색상 변경 또는 밝기 조정

모든 버튼:
  - 비활성 상태 명확히 표시 (회색, 투명도)
  - 활성 상태와 시각적 차이

모든 입력 필드:
  - 포커스 시 테두리 색상 변경
  - 오류 시 빨간 테두리
  - 플레이스홀더 텍스트 제공
14.3 스크린 리더 지원
모든 중요 요소에 alt 텍스트:
  - 버튼: "단어 추가", "시험 시작"
  - 아이콘: "즐겨찾기", "삭제"
  - 이미지: 설명 제공

진행 상황 알림:
  - "5 / 50 문제 진행 중"
  - "학습 완료, 정답률 85%"

15. 성능 최적화 고려사항
15.1 대용량 데이터 처리
단어 목록 (1,000개 이상):
  - 가상 스크롤링 (Virtual Scrolling)
  - 한 번에 100개씩만 렌더링
  - 스크롤 시 동적 로드/언로드

검색:
  - 300ms 디바운스
  - 백그라운드 스레드에서 처리
  - 결과 캐싱
15.2 애니메이션 최적화

  - CSS transforms 사용

애니메이션 프레임 레이트:
  - 목표: 60fps
  - 부드러운 전환 보장
15.3 메모리 관리
탭 전환 시:

  - 필요할 때만 데이터 로드

  - 사용 후 메모리 해제
