# 단어장 프로그램 기능 명세서 (수정본)

**작성일:** 2025년 9월 29일  
**버전:** 2.0  
**문서 유형:** 기능 목록 및 요구사항 정리

---

## 1. 단어 암기 (플래시카드)

### 1.1 학습 모드 선택
**영→한 모드**
- 영어 단어를 보고 한국어 뜻 맞추기
- 단어 카드 형태로 표시

**한→영 모드**
- 한국어 뜻을 보고 영어 단어 맞추기
- 철자 정확도까지 체크

**양방향 혼합 모드** (추가 제안)
- 영→한과 한→영을 무작위로 섞어서 학습
- 실전 감각 향상

### 1.2 출제 순서 방식
**순차 모드**
- 단어장에 등록된 순서대로 학습
- 체계적인 학습에 적합

**랜덤 모드**
- 완전 무작위 순서로 출제
- 암기 순서에 의존하지 않도록 훈련

**개인화 모드** ⭐
- 오답률이 높은 단어의 출제 비중 자동 증가
- 가중치 알고리즘 적용 (예: 오답률 70% 이상 단어는 3배 더 자주 출제)
- 최근 학습 이력 반영하여 약한 부분 집중 공략

### 1.3 답안 표시 방식
**블라인드 모드**
- 답을 가린 채로 먼저 생각해보기
- 능동적 회상(Active Recall) 효과

**노출 모드**
- 답을 함께 표시하여 빠른 확인 학습
- 복습 단계에서 유용

**힌트 모드** (추가 제안)
- 첫 글자만 표시
- 단어 길이(언더바) 표시
- 점진적 힌트 제공

### 1.4 시각적 피드백
**오답률 기반 색상 표시**
- 🔴 빨강: 오답률 70% 이상 (집중 학습 필요)
- 🟠 주황: 오답률 40-70% (주의 필요)
- 🟢 초록: 오답률 40% 미만 (양호)
- ⚪ 회색: 학습 이력 없음 (신규)

**학습 진행 표시**
- 현재 카드 번호 / 전체 카드 수
- 진행률 바(Progress Bar)
- 남은 카드 수

**즉시 피드백**
- 정답/오답 즉시 표시
- 정답 시: ✓ 표시와 함께 초록색 배경
- 오답 시: ✗ 표시와 함께 올바른 답 제시

### 1.5 학습 중 기능
**즐겨찾기**
- 학습 중 바로 별표 아이콘으로 즐겨찾기 등록/해제
- 나중에 집중 복습용

**건너뛰기** (추가 제안)
- 현재 모르는 단어는 나중에 다시 보기
- 건너뛴 카드는 세션 마지막에 다시 출제

**예문 보기** (추가 제안)
- 단어 활용 예문 팝업
- 문맥 속에서 단어 이해

## 2. 단어 관리

### 2.1 단어 목록 조회
**전체 단어 리스트**
- 등록된 모든 단어를 테이블 형태로 표시
- 영어, 한국어, 오답률, 마지막 학습일, 즐겨찾기 여부 등 표시

**검색 기능** (추가 제안)
- 영어 단어로 검색
- 한국어 뜻으로 검색
- 부분 일치 검색 지원

**필터링** (추가 제안)
- 오답률별 필터 (취약/주의/양호)
- 즐겨찾기만 보기
- 학습 안 한 단어만 보기
- 최근 수정한 단어만 보기

**정렬** (추가 제안)
- 알파벳 순
- 오답률 높은 순
- 최근 학습일 순
- 추가한 날짜 순

### 2.2 단어 데이터 입출력
**단어 업로드 (CSV 형식)**
- 파일 형식: `영어, 한국어, 메모(선택)` 컬럼
- 예시: `apple, 사과, 과일 종류`
- 중복 단어 자동 감지 및 선택 (추가 제안)
  - 기존 단어 유지
  - 새 단어로 덮어쓰기
  - 병합 (메모 합치기)
- 대량 업로드 지원 (1,000개 이상)

**단어 다운로드 (CSV 형식)**
- 전체 단어장 내보내기
- 필터링된 단어만 내보내기
- 학습 데이터 포함 옵션 (오답률, 학습 이력 등)

### 2.3 단어 편집
**단어 추가**
- 개별 단어 직접 입력 등록
- 영어, 한국어, 메모 입력

**단어 수정**
- 기존 단어의 영어/한국어 뜻 변경
- 메모 추가/수정
- 수정 날짜 자동 기록

**단어 삭제**
- 개별 단어 삭제
- 일괄 선택 삭제 (추가 제안)
- 삭제 확인 메시지

### 2.4 메모 및 수정 이력
**개인 메모 기능**
- 각 단어에 자유롭게 메모 작성
- 예: "어원: apple은 라틴어에서 유래", "헷갈리는 단어: apply와 구분"
- 메모는 플래시카드에서도 확인 가능 (추가 제안)

**수정 이력 추적**
- 최종 수정 날짜 표시
- 수정된 단어 목록 별도 탭에서 확인
- 수정 전/후 비교 보기 (추가 제안)

### 2.5 오답 노트 자동 생성 및 복습
**시험 후 오답 노트 자동 생성** ⭐
- 시험에서 틀린 단어들을 자동으로 "오답 노트" 컬렉션으로 저장
- 오답 노트는 별도 탭에서 관리

**복습 모드 연결**
- 오답 노트 단어들만 모아서 플래시카드 복습
- "오답 노트 복습하기" 버튼으로 즉시 시작
- 복습 후 다시 틀린 단어는 계속 오답 노트에 유지

**오답 노트 관리** (추가 제안)
- 완전히 숙지한 단어는 오답 노트에서 제거
- 오답 노트별 생성 날짜 표시
- 여러 시험의 오답 노트 통합 보기

### 2.6 답변 제한 시간 설정
**플래시카드 제한 시간**
- 카드당 답변 시간 설정 (예: 5초, 10초, 15초, 무제한)
- 시간 초과 시 자동으로 다음 카드로 넘어감
- 제한 시간 카운트다운 표시

**시험 제한 시간**
- 전체 시험 시간 제한 (예: 20문제에 10분)
- 문항당 시간 제한 (예: 문제당 30초)
- 시간 종료 시 자동 제출

## 3. 단어 시험

### 3.1 시험 설정
**출제 범위 선택**
- 전체 단어장
- 특정 범위 (예: 1~100번 단어)
- 즐겨찾기 단어만
- 오답률 높은 단어만
- 오답 노트에서 선택 (추가)

**문항 수 설정**
- 사용자 지정 (예: 10문제, 20문제, 50문제)
- 전체 단어 모두

**제한 시간 설정** (2.6과 연계)
- 전체 시험 제한 시간
- 문항당 제한 시간
- 무제한 모드

### 3.2 출제 순서
**랜덤 순서**
- 완전 무작위 출제
- 매번 다른 순서로 출제

**개인화 순서** ⭐
- 오답률 높은 단어 우선 출제
- 최근 학습하지 않은 단어 포함
- 가중치 기반 출제 알고리즘

**순차 순서** (추가 제안)
- 단어장 등록 순서대로 출제
- 체계적인 평가용

### 3.3 문제 유형

#### 3.3.1 단답형
**영→한 단답형**
- 영어 단어 보고 한국어 입력
- 텍스트 입력 후 엔터키로 제출

**한→영 단답형**
- 한국어 뜻 보고 영어 단어 입력
- 대소문자 구분 안 함 (추가 제안)
- 유사 답안 처리 (추가 제안)
  - 오타 1-2자 허용 옵션
  - 복수 정답 인정 (예: "colour" = "color")

#### 3.3.2 객관식
**4지선다 / 5지선다**
- 설정에서 선택지 개수 선택 가능

**오답 선택지 생성 방식** (추가 제안)
- 유사한 뜻의 다른 단어
- 유사한 철자의 단어
- 완전 랜덤 단어
- 혼합 방식

**문제 유형**
- 영어 보고 한국어 선택
- 한국어 보고 영어 선택

### 3.4 시험 진행
**진행 상황 표시**
- 현재 문제 번호 / 전체 문제 수
- 진행률 바
- 남은 시간 (제한 시간 설정 시)

**문제 이동** (추가 제안)
- 이전 문제로 돌아가기
- 다음 문제로 건너뛰기
- 답안 수정 가능

**임시 저장** (추가 제안)
- 시험 중 중단 시 임시 저장
- 나중에 이어서 계속하기

### 3.5 시험 결과
**점수 및 통계**
- 정답 개수 / 전체 문항 수
- 정답률 백분율
- 소요 시간
- 평균 문항당 시간

**오답 분석**
- 틀린 문제 목록 상세 표시
- 내가 입력한 답 vs 정답 비교
- 왜 틀렸는지 확인

**오답 노트 자동 생성** (2.5와 연계)
- 틀린 단어들을 자동으로 오답 노트에 추가
- 바로 복습하기 버튼 제공

**시험 이력 저장** (추가 제안)
- 과거 시험 결과 조회
- 시험별 점수 변화 추이 확인
- 시험 날짜, 범위, 점수 기록

**성적 추이 그래프** (추가 제안)
- 시간에 따른 점수 변화 그래프
- 정답률 추세 분석

## 4. 학습 관리

### 4.1 학습 목표 설정
**일일 목표**
- 하루 학습할 단어 개수 설정 (예: 하루 50개)
- 하루 학습 시간 목표 (예: 하루 30분) (추가 제안)
- 목표 달성률 실시간 표시

**주간/월간 목표** (향후 기능에서 추가됨)
- 주간 학습 단어 수 목표
- 월간 목표 설정

**특정 시험 D-Day 연동** (향후 기능에서 추가됨)
- 시험 날짜 설정 (예: 수능 D-100)
- D-Day까지 필요한 일일 학습량 자동 계산
- 진도율 표시

**학습 알림** (추가 제안)
- 설정한 시간에 학습 리마인더
- 목표 미달성 시 알림
- 연속 학습일 유지 알림

### 4.2 학습 통계 - 오늘 통계
**오늘의 학습 기록**
- 오늘 학습한 단어 수
- 오늘 학습 시간
- 오늘 정답률
- 오늘 목표 달성률
- 오늘 시험 본 횟수 및 평균 점수

**오늘의 하이라이트** (추가 제안)
- 오늘 처음 맞춘 어려운 단어
- 오늘 가장 많이 본 단어
- 오늘의 학습 시간대

### 4.3 학습 통계 - 전체 통계
**누적 학습 데이터**
- 총 학습 단어 수 (고유 단어)
- 총 학습 시간
- 평균 정답률
- 총 시험 횟수 및 평균 점수

**단어 숙지도**
- 완전 숙지 (정답률 90% 이상): X개
- 어느 정도 숙지 (정답률 70-90%): X개
- 취약 (정답률 70% 미만): X개
- 미학습: X개

**취약점 분석**
- 가장 많이 틀린 단어 TOP 10
- 오답률 높은 단어 목록
- 자주 헷갈리는 단어 쌍 (추가 제안)

**학습 성과**
- 연속 학습일 (스트릭)
- 최장 연속 학습 기록
- 주간/월간 평균 학습량

**통계 시각화** (향후 기능에서 추가됨)
- 일별/주별/월별 학습량 그래프
- 정답률 변화 추이 그래프
- 학습 시간대 히트맵
- 단어 숙지도 원형 차트

### 4.4 학습 분석 및 피드백 (추가 제안)
**학습 패턴 분석**
- 가장 효율적인 학습 시간대
- 요일별 학습 패턴
- 학습 세션 최적 길이

**개인 맞춤 추천**
- 오늘 복습해야 할 단어 제안
- 취약한 단어 집중 학습 제안
- 목표 달성을 위한 일일 권장 학습량

**주간/월간 리포트** (추가 제안)
- 주간 학습 요약 리포트 생성
- 월간 성장 리포트
- 개선 사항 및 칭찬 메시지

## 5. 공통 기능

### 5.1 즐겨찾기 ⭐
**즐겨찾기 추가/해제**
- 플래시카드 학습 중: 별표 아이콘 클릭
- 단어 관리 화면: 단어 목록에서 별표 클릭
- 시험 화면: 시험 후 결과 화면에서 특정 단어 즐겨찾기 (추가 제안)

**즐겨찾기 목록 관리**
- 별도 탭에서 즐겨찾기한 단어만 모아보기
- 즐겨찾기 메모 추가 (왜 즐겨찾기했는지)
- 즐겨찾기 해제

**즐겨찾기 활용**
- 즐겨찾기 단어만으로 플래시카드 학습
- 즐겨찾기 단어만으로 시험
- 즐겨찾기 단어 CSV 내보내기

### 5.2 개인화 (자주 틀리는 단어 관리) ⭐⭐
**핵심 개인화 알고리즘**

이 기능은 프로그램의 핵심 차별화 요소입니다.

**데이터 수집**
- 각 단어마다 정답 횟수, 오답 횟수 기록
- 마지막 학습 날짜 기록
- 학습 시도 총 횟수 기록

**오답률 계산**
- 오답률 = (오답 횟수) / (총 학습 횟수) × 100
- 최근 학습에 더 높은 가중치 부여 (추가 제안)
- 최소 학습 횟수 (예: 3회) 미만인 경우 신규 단어로 분류

**개인화 적용**

1. **플래시카드 학습 시**
   - 오답률 70% 이상 단어: 출제 확률 3배
   - 오답률 40-70% 단어: 출제 확률 2배
   - 오답률 40% 미만 단어: 출제 확률 1배
   - 신규 단어: 출제 확률 1.5배

2. **시험 출제 시**

### 7.2 추가 보완이 필요한 부분 🔧

#### 7.2.1 우선순위 높음 - 즉시 명확화 필요

**1. 간격 반복 학습 알고리즘 구체화 필요**
현재 "개인화"라는 표현만 있고, 구체적인 알고리즘이 명시되지 않았습니다.

**보완 제안:**
- 복습 주기 계산 방식 정의 필요
  - 정답 시: 다음 복습일 = 현재일 + (이전 간격 × 2.5)
  - 오답 시: 다음 복습일 = 현재일 + 1일
- "숙지 완료" 판단 기준
  - 예: 연속 5회 정답 + 마지막 학습 간격 30일 이상
- 복습 우선순위 점수 계산
  - 점수 = 오답률 × 0.6 + 복습 지연일수 × 0.4

**2. 데이터 구조 설계**
각 단어마다 어떤 정보를 저장할지 명확히 해야 합니다.

**보완 제안:**
```
단어 테이블:
- word_id (고유 ID)
- english (영어 단어)
- korean (한국어 뜻)
- memo (개인 메모)
- category_id (카테고리)
- created_date (생성일)
- modified_date (수정일)
- is_favorite (즐겨찾기 여부)

학습 이력 테이블:
- history_id
- word_id
- study_date (학습 날짜)
- is_correct (정답 여부)
- response_time (응답 시간)
- study_type (플래시카드/시험)

통계 테이블:
- word_id
- total_attempts (총 시도 횟수)
- correct_count (정답 횟수)
- wrong_count (오답 횟수)
- last_study_date (마지막 학습일)
- next_review_date (다음 복습 예정일)
- mastery_level (숙지도 0-5)
```

**3. "개인화" 가중치 구체화**
현재 "많이 틀리는 문제의 비중 상승"이라고만 되어 있습니다.

**보완 제안:**
```python
def calculate_word_weight(wrong_rate, days_since_study):
    base_weight = 1.0
    
    # 오답률 기반 가중치
    if wrong_rate >= 0.7:
        wrong_weight = 3.0
    elif wrong_rate >= 0.4:
        wrong_weight = 2.0
    else:
        wrong_weight = 1.0
    
    # 복습 지연 가중치
    if days_since_study >= 7:
        delay_weight = 2.0
    elif days_since_study >= 3:
        delay_weight = 1.5
    else:
        delay_weight = 1.0
    
    return base_weight * wrong_weight * delay_weight
```

#### 7.2.2 우선순위 중간 - MVP 이후 구체화

**1. 유사 답안 인정 로직**
단답형에서 "오타 허용"이 언급되었지만 구체적이지 않습니다.

**보완 제안:**
- Levenshtein Distance 알고리즘 사용
- 편집 거리 2 이내는 오타로 인정
- 예: "apple" 입력 시 "aple", "appel" 정답 인정

**2. 객관식 오답 선택지 생성 알고리즘**
어떻게 그럴듯한 오답을 만들지 명확하지 않습니다.

**보완 제안:**
- 방법 1: 유사한 뜻의 단어 (동의어 사전 활용)
- 방법 2: 같은 카테고리 내 다른 단어
- 방법 3: 철자가 비슷한 단어 (첫 글자 같은 단어)
- 방법 4: 최근 학습한 단어 중 랜덤

**3. 학습 통계 계산 방식**
"평균 정답률", "학습 완료율" 등의 계산 방식이 명확하지 않습니다.

**보완 제안:**
```
평균 정답률 = Σ(각 단어의 정답 횟수) / Σ(각 단어의 총 시도 횟수)

학습 완료율 = (숙지 완료 단어 수) / (전체 단어 수) × 100

숙지 완료 기준 = 정답률 90% 이상 AND 최근 3회 연속 정답
```

```
   - 틀린 단어만 모아서 복습
   ↓
7. 복습 주기에 따라 자동 알림
   - "오늘 복습할 단어 20개"
```

#### 데이터 흐름
```
단어 입력 (수동 또는 CSV 업로드)
   ↓
단어 데이터베이스 저장
   ↓
학습 시작
   ↓
학습 이력 기록 (정답/오답, 시간)
   ↓
통계 계산 (오답률, 복습일)
   ↓
개인화 알고리즘 적용
   ↓
다음 학습 시 반영
```

### 7.5 위험 요소 및 대응 방안

- **위험**: 간격 반복 알고리즘이 복잡하여 구현 어려움
- **대응**:
  - MVP에서는 간단한 Leitner 시스템 사용
  - 이후 버전에서 SM-2로 업그레이드
  - 단계별 구현 및 테스트

**사용자 데이터 손실 위험**
- **위험**: 프로그램 오류로 학습 데이터 손실
- **대응**:
  - 자동 백업 기능 (일일 백업)
  - SQLite 트랜잭션 사용
  - 복구 기능 제공

**학습 지속성 위험**
- **위험**: 사용자가 며칠 사용 후 포기
- **대응**:
  - 학습 동기 부여 시스템 (스트릭, 뱃지)
  - 목표 달성 시 축하 메시지
  - 간단한 일일 루틴 (5분 빠른 복습)
